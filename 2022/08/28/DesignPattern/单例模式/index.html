单例模式

饿汉式

基础形式

静态代码块形式

优点：

  • 线程安全

缺点

  • 如果没有使用的话，会造成资源浪费

懒汉式

优点：

  • 启用了 Lazy Loading，不会浪费资源，当需要时才创建实例，否则不创建，但是只能单线程

缺点：

  • 线程不安全

线程安全懒汉式

通过 synchronized 上锁实现

反例：同步代码块(不能解决线程安全问题)

  • 原因：当有第一个线程进入 if 判断语句后，如果还没有获取 synchronized 锁，又进来一个线程，这时 if 语句中就存在了两个线程，因为 synchronized 锁的是 Singleton 这个类，因此线程只能按顺序执行，第一个线程执行完 new 操作后，释放锁，这时第二个线程又执行 new 操作，这样就出现了两个对象，因此不能保证是同一个对象，因此线程不安全

DoubleCheck

  • 相较于上面的同步代码块，在创建实例前添加了一个 if 判断，这样就可以保证即使是多个线程，在第一个线程创建完 Singleton 对象后，第二个线程进行操作时，先判断是否已经创建好了对象，判断到有其他线程已经创建完对象后，直接返回已经创建好的对象，因此线程安全

静态内部类

  • 借助了 JVM 类加载机制中，静态内部类如果没有被使用不会被加载的原则，当需要对象时，在加载静态内部类，又由于是静态的，只会加载一次，后面在加载也是使用第一次加载创建的对象，因此保证了线程安全

枚举类

不仅能解决多线程问题，还能解决反序列化重新创建新的对象
