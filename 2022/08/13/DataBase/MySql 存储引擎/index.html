MySql 存储引擎

索引 Index 是在存储引擎层实现的

存储引擎就是存储数据、建立索引、更新/查询数据等技术的实现方式。他是基于表而非基于库的，所以也可以被称为表类型

表引擎可以通过 sql 语句来查看

默认引擎为 InnoDB

查看所有引擎可以通过 sql 语句来查询

如何指定表存储引擎

InnoDB

是一种高可靠性和高性能的存储引擎，MySQL 5.5 之后，为默认

特点

  • DML (insert, delete, update 语句) 操作遵循 ACID 模型 ，支持事务
  • 行级锁，提高并发访问性能，
  • 支持外键

文件

  • 存储的磁盘文件为 xxx.idb : xxx 为表名，每一个文件都对应了一个表空间文件、内包含了存储在该表的表结构、数据和索引

选择：

  • 需要事务、外键、并发，除了插入与查询外，还要大量的删改，选择

Innodb 的逻辑存储结构

TableSpace : 表空间

Segment : 段

Extent : 区

Page : 页

Row : 行

MyISAM

MySQL 早期的存储引擎

特点

  • 不支持事务也不知外键
  • 支持表锁，不支持行锁
  • 访问快

文件

  • xxx.sdi : 表结构信息
  • xxx.MYD : 存储数据
  • xxx.MYI : 存储索引

选择：

  • 以读和写为主，少量删改，不需要高并发，可选

Memory

存储在内存中，但受制于硬件以及电源影响，只能作为临时表或缓存使用

特点

  • 内存存放
  • 支持 Hash 索引

文件

  • xxx.sdi 表结构信息

选择

  • 作为缓存使用，可选
