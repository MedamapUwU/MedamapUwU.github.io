Web 域对象

四个域的作用范围大小：

PageContext (page 域) < Request < Session < ServletContext (applictaion 域)

PageContext

  • 作用范围：
    • 当前的整个 JSP 页面，当请求超过当前页面就不可使用了，因此不能向其他页面传递信息
    • 每次请求分别对应不同的 request 。
  • 生命周期：
    • 当 JSP 页面被执行（即请求时）开始，响应结束时销毁，生命周期结束

Request

  • 作用范围：
    • 当前请求的整个请求链。
  • 生命周期：
    • 当发送请求时，生命周期开始，服务器返回响应后，请求结束，生命周期结束。

Session

  • 作用范围
    • 当前会话
  • 生命周期
    • 在第一次调用 request.getSession() 方法时，服务器会检查是否已经有对应的session,如果没有就在内存 中创建一个 session 并返回。
    • 当一段时间内 session 没有被使用（默认为30分钟），则服务器会销毁该 session。
    • 如果服务器非正常关闭（强行关闭），没有到期的 session 也会跟着销毁。
    • 如果调用 session 提供的 invalidate()，可以立即销毁 session 。
    • 注意：服务器正常关闭，再启动，Session 对象会进行钝化和活化操作。同时如果服务器钝化的时间在 session 默认销毁时间之内，则活化后 session 还是存在的。否则 Session 不存在。如果JavaBean 数据在 session 钝化时，没有实现 Serializable 则当 Session 活化时，会消失。

ServletContext

  • 作用范围：
    • 整个 Web 应用
  • 生命周期：
    • 当 Web 应用被加载进容器时创建代表整个 web 应用的ServletContext 对象，当服务器关闭或 Web 应用被移除时，ServletContext 对象跟着销毁。
