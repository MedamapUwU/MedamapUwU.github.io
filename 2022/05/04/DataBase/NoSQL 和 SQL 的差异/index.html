NoSQL 与 SQL 的差异

NoSQL 与 SQL 的差异

  • 结构

    • SQL 采取的是结构化的形式，即在设计之初就确定好每个字段的类型、长度。如果在数据量很多的情况下，再次修改表的结构可能会导致很严重的业务瘫痪，因此添加字段的难度较大，不够灵活
    • NoSQL 采用的是非结构化的形式(Key-Value) 的形式,存储的形式多种多样，因为没有锁死字段的各项信息，因此后期修改的成本低，影响相较于 SQL 数据库来说小。
      • 键值类型 (Redis)
      • 文档类型 (MongoDB)
      • 列类型 (HBase)
      • Graph 类型 (Neo4j)

  • 关联

    • SQL 为关系型数据库，存在表与表之间的联系

    • NoSQL 为非关系型数据库，没有表之间的关系，可以通过 json 格式，相互嵌套来表示他们之间的关系。

      • 但是这样会造成数据之间的重复，即：不同用户购买同一件商品，保存的数据内会出现相同的格式。

  • SQL 查询

    • SQL 可以通过 SQL 语句来进行对表的增删改查
    • NoSQL 不通过 SQL 语句来查询数据库，可以通过不同的方式来查询
      • Redis ：采用命令的方式 get user: 1
      • MongoDB ：采用函数的方式 db.users.find({_id:1})
      • elasticsearch：采用请求的方式 GET http://localhost:9200/users/1

  • 事务

    • SQL 中的 增删改 都会有事务的需求，既需要满足 ACID
    • NoSQL 中需要满足 BASE 即可。

  • 存储

    • SQL 于 磁盘 存储
    • NoSQL 于 内存 存储

  • 扩展性

    • SQL 垂直扩展
    • NoSQL 水平扩展
