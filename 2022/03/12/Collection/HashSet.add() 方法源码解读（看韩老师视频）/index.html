源码
第一个 if

我们挨个分析

很明显，这是一个创建变量的过程

  • tab 为一个 Node 数组
  • p 为一个 Node 节点
  • n 从 n = tab.length 可以看出这个变量是存放 tab.length 的，即 node 数组的长度
  • i 从 i = (n - 1) & hash 可以看出这个变量是存放通过 & 运算算出来的最终位置

接下来第一个 if 语句

这里我们看到了一个没有提到的参数 table

从上方我们可以得出 table 其实也是一个 node 数组，transient 的用法我不是很清楚 OwO

ok，我们开始分析内部条件

  • (tab = table) == null
    • 首先将 table 赋给 tab，然后进行比较是不是 null
    • 说明这是在判断 tab 是否没有指向任何对象
  • (n = tab.length) == 0
    • 将 tab 的 length 赋值给变量 n ，然后判断是否为 0
    • 说明这是在判断 tab 创建了一个对象后，长度是否为 0
  • ||
    • 短路或，有一个满足即可，如果第一个不满足，则不会进行判断第二个
  • n = (tab = resize()).length;
    • 这里调用了一个 resize() 方法，赋值给 tab ，然后求出长度赋值给 n
resize 方法

让我们进入这个 resize() 方法，一探究竟

... 的部分我们暂时不做讨论（其实我并没有看太懂，大概就是对于上述确定好容量后的创建扩容）

我们来看一下是如何确定 node 数组的大小的

  • Node<K,V>[] oldTab = table;

    • 将 table 赋给 oldTab , 没有什么好说的

  • int oldCap = (oldTab == null) ? 0 : oldTab.length;

    • oldCap 是一个用来存放 oldTab 的 length 长度的变量
    • 使用了三元运算符，判断 oldTab 是否为未指向任何对象，是，则返回 0 作为 oldCap 的长度返回；否则，返回 oldTab.length

  • int oldThr = threshold;

    • oldThr 变量是用来存放数组扩容的临界值的，threshold 是用来存放这个临界值的

  • int newCap, newThr = 0;

    • newCap 用来存放新数组的大小
    • newThr 用来存放新的临界值

接下来我们来看判断语句的部分

if (oldCap > 0)

第一个 if 部分，首先判断 oldCap 是否大于 0 ，即判断 oldTab 内是否有空间

有空间则进行下一步的判断

  • if (oldCap >= MAXIMUM_CAPACITY)

  • 如果大于 MAXIMUM_CAPACITY （2^30^），将临界值 threshold 设为 Integer 的最大值

  • 直接返回 旧表 (oldTab)

  • else if ((newCap = oldCap << 1) < MAXIMUM_CAPACITY && oldCap >= DEFAULT_INITIAL_CAPACITY)

    • 首先将 旧空间大小 (oldCap) 左移一位，即 oldCap * 2 ，然后赋值给 新空间大小 (newCap) ,判断是否小于 2^30^,
    • 判断 oldCap 是否大于等于空间的初始值 DEFAULT_INITIAL_CAPACITY （2^4^）
    • 用短路与 (&&) 连接，如果第一个条件不满足，则直接跳过，不进行第二个条件的判断
    • 将 旧临界值 (oldThr) 左移一位，即 oldThr * 2 ，并且赋值给 newThr成为新的临界阈值

else if (oldThr > 0)

如果不满足 旧空间大小 (oldCap) > 0 ，即没有被开辟空间，则判断 旧临界值 (oldThr) 是否大于 0 ，即初始化是否被初始化

初始化后，则将初始化的 threshold 的值赋给 新空间大小 (newCap) ，即新容量

else

如果不满足前两个，没有完成任何初始化，即一个新对象，则进行默认初始化

将默认初始化空间赋值给 新空间大小 newCap

将 装填因子 (*) 默认初始化空间的积强转为 int 赋值给 新临界值 (newThr) 作为新的临界值

if (newThr == 0)

判断新临界值是否等于 0 ，即没有被初始化过

则将 新空间大小 (newCap) 乘上 (*) 装填因子 (loadfactor) ，强转为 float 作为 新临界值 (newThr) 的预选值 ft

然后通过比较 新空间大小 (newCap) 是否小于 最大空间 (MAXIMUM_CAPACITY) 短路与 (&&) ft 是否小于强转为 float 的 MAXIMUM_CAPACITY ，是，则将 ft 强转为 int 后赋值给 新临界值 (newThr) ; 否则，将 Integer.MAX_VALUE 当做 新临界值

做完上述操作之后，进行赋值操作

  • threshold = newThr
    • 将 newThr 的值赋值给 thrshold ，即初始化，这也是前面判断是否初始化的源头
  • Node<K,V>[] newTab = (Node<K,V>[])new Node[newCap];
    • 创建一个新的 node 数组，大小为 newCap
  • table = newTab;
    • 将 newTab 的赋给 table ，即 table 也完成了初始化

接下来就是创建新空间的操作内容，最后 return newTab;

至此，resize 方法结束，其实这就是一个数组扩容的方法，它确定了数组的空间大小和进行再次扩容的临界值。

第二个 if

我们接着分析接下来的语句，第二个 if 语句

分析条件，首先从内部开始往外分析

  • i = (n - 1) & hash
    • 很明显 i 是位置变量，这个表达式是为了通过 (n - 1) & hash 计算出一个位置，来放置元素
  • p = tab[i]
    • 将 tab 表中的 i 位置的元素赋给 p
  • if(p == null)
    • 判断该位置是否为空
  • tab[i] = newNode(hash, key, value, null);
    • 为空则将该对象放置到 tab[i] 的位置
else

  • Node<K,V> e; K k;

    • e 为 一个 node 对象
    • k 为一个 K 类型的元素，K 即为 add 方法传入的参数的参数类型

  • if (p.hash == hash && ((k = p.key) == key || (key != null && key.equals(k)))) {e = p};

    • p.hash == hash (1)

      • 这个条件是判断当 p 位置已经存在一个元素，并且这个元素的 hash 与 要插入的元素的 hash 相同时的情况

    • (k = p.key) == key (2)

      • 将 p 的值 (key) 赋给 k

      • 判断 p 的 key 值是否与要插入的元素值 key 是否相同，即查看是否值相同

    • (key != null && key.equals(k)) (3)

      • key 不为空
      • && 短路与
      • key.equals(k) 判断两个对象是否相同，主要应对那些重写了 equals 方法的类，来判断是否相同

    • (1) && ((2) || (3))

      • (2) 和 (3) 二选一即可，即存放的和要插入的 key 值相同
      • 存放的 hash 与 要插入的 hash 值相同
      • 即这个条件是为了判断要插入的元素和已经存在的对象是否为同一个对象

    • e == p

      • 如果是同一个对象，则将 p 赋给 e

  • else if (p instanceof TreeNode) {e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);}

    • 这涉及到树化，暂时不讨论

  • else

  • for (int binCount = 0; ; ++binCount)
    • 我们需要注意的一点是，这个循环是没有判断结束的条件的 int binCount = 0; //没有判断结束的条件 ; ++binCount，也就是说，这个循环是一个无限循环，跳出循环的唯一途径就是内部语句的 break 跳出来
  • if ((e = p.next) == null)
    • 将 p.next 赋给 e ，即将 p 节点指向的下一个节点赋给 e
    • e == null 判断 p 的下一个节点是否为空
  • p.next = newNode(hash, key, value, null);
    • 为空，则将要插入的元素插入在 e 的位置
  • if (binCount >= TREEIFY_THRESHOLD - 1)
    • binCount 是计算链的长度的计数器，如果链的长度 >= 8 （TREEIFY_THRESHOLD - 1 的值为 7 ，但是 binCount 是从 0 开始计数的，所以到 7 时，长度为 8），则会执行 treeifyBin(tab, hash); 进行红黑树化，具体内部不做讨论
    • break
      • 执行完该 if 的内容，即已经插入了新元素后，break退出
  • if (e.hash == hash && ((k = e.key) == key || (key != null && key.equals(k))))
    • 上方已经讲过，检测是否为同一对象，是则直接 break
  • p = e
    • 将 e 赋值给 p , 即将 p.next 赋给 p ，即将 p 向后移一个节点的位置

然后一直循环到 break 为止

退出后，执行下一个 if

  • 并没有看太懂
  • 可以看看这位大佬写的文章

CSDN 上一位大佬讲的 putValue 方法

最后的部分
  • ++modCount
    • 成功添加后，modCount ++，计数器
  • if (++size > threshold)
    • 如果数组内的元素的数量 > 初始临界值，则进行 resize
  • afterNodeInsertion(evict);
    • 这个方法是留给 HashMap 的子类来进行排序或其他操作
  • return null
    • 返回 null 值

put 方法返回了 null 值，对于 add 方法，判断了 put 方法返回值为 null 返回值为 true ，则表示添加成功
