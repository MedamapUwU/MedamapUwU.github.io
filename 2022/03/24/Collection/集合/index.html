Java 集合接口和实现分离

何为将接口和实现分离？以队列 (Queue) 来举例：

队列接口可以写为

这个接口并没有说明队列这个数据结构具体是如何实现的。

通常是有两种实现方法：

  • 循环数组：较链表效率更高，但有界集合，容量有限
  • 链表：较循环数组效率较低，但是容量无上限

实际 Java 类库里面没有 CircularArrayQuene<E> 和 LinkedListQuene<E> 类，只是用来解释

接口并不给出具体的实现方法，只给出一些通用的方法，具体的操作原理通过被实现的类或接口来完成

声明时也可以利用这个特性来根据不同的情况作出应对:

当在程序中需要使用队列时，只要构造了集合，就不需要知道使用了哪种实现，在构造集合对象时，才使用具体的类

当需要更改对象的类型时，只需要改后面的部分就可以了
