Windows 中 Nginx 解压出问题导致不能运行 Nginx

正常的目录情况

有问题的解压情况

可以发现，少了 temp 与 logs 目录， 导致点击 nginx.exe 或在 cmd 中启动无响应

因此只需要手动创建 temp 目录 与 logs 目录即可解决这个问题

除此之外，在未修改端口号的情况下 80 端口是被默认占用的

因此，记得在 conf 目录下的 nginx.config 中修改为一个未被占用的端口号即可

启动成功后，在任务管理器处会出现两个 nginx.exe 的服务

这时访问相应端口的页面，就会出现下面的情况

现在即可使用 Nginx 了

要退出使用 Nginx，需要在 cmd 处使用命令 nginx.exe -s stop 命令停止
