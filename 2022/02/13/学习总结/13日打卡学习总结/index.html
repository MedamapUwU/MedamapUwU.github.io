收获：

方法参数：

将参数传递给方法的两种方式：

  • 按值调用 (call by value)：表示方法接收的是调用者提供的值
  • 按引用调用 (call by reference)：表示方法接收的是调用者提供的变量地址

Java 程序语言总是采用按值调用，也就是说，方法得到的是所有参数值的一个副本

方法参数有两种类型

  • 基本数据类型（数字、布尔值）
  • 对象引用

如果是按引用调用，那么 x 就应该是 mike ，y 就应该是 tom ，相应的指向应该变化，打印输出的结果应为 After : mike = Tom After : tom = Mike

但是输出结果为 After : mike = Mike After : tom = Tom 因此也证明了是按值调用而非按引用调用

因此，方法得到的是所有参数值的一个副本，而非本身

类设计的技巧

  • 一定保证数据私有

  • 一定要对数据初始化

  • 不要在类中使用过多的基本类型

    • 这个想法是要用其他的类替换使用多个相关的基本类型。这样使类更易于理解，也易于修改
    • 例如：用一个名为 Address 的新类替换一个 Customer 类中以下的实例字段
  • 不是所有的字段都需要单独的字段访问器和修改器
  • 分解有过多指责的类
    • 如果明显的可以将一个复杂的类分解成两个更为简单的类，就应该将其分解、
    • 反面教材
  • 类名和方法名能够体现他们的职责
  • 优先使用不可变的类

理解方法调用

下面假设要调用 x.f(args) ，隐式参数 x 声明为类 C 的一个对象，下面是调用过程详解

  1. 编译器查看对象的声明类型和方法名。

    有可能存在多个方法名为 f 的重载方法，编译器将会一一列举 C 类中所有名为 f 的方法和其超类中所有名为 f 且可以访问的方法（超类的私有方法不可访问）

    至此，编译器已经知道所有可能被调用的候选方法

  2. 编译器要确定方法调用中提供的参数类型。

    如果在所有名为 f 中的方法中，存在一个与提供的参数类型完全匹配的方法，那么就选择这个方法，这个过程被称为 重载解析

    如果编译器没有找到与参数类型匹配的方法，或者发现经过类型转换后有多个方法与之匹配，编译器就会报错

    如果在子类中定义了一个与超类签名相同的方法，那么子类中的这个方法就会覆盖超类中这个相同签名的方法

    返回类型不是签名的一部分，不过在覆盖一个方法时，需要保证返回类型的兼容性。允许子类将覆盖方法的返回类型改为原返回类型的子类型

  1. 如果是 private 方法、 static 方法、 final 方法或者构造器，那么编译器讲可以准确地知道应该调用哪个方法，这称为 静态绑定，如果要调用的方法依赖于隐式参数的实际类型，那么必须在运行时使用动态绑定

  2. 程序运行且采用动态绑定调用方法时，虚拟机必须调用与 x 所引用对象的是实际类型对应的那个方法，现在子类找匹配的方法，没有的话去超类寻找

由于每次调用都要完成这个流程时间开销巨大，因此，虚拟机预先为每个类计算了一个方法表，其中列出了所有方法的签名和要调用的实际方法。真正调用时，直接查表即可

如果要调用是 super.f(param), 那么编译器将对隐式参数超类的方法表进行搜索

动态绑定的特性：

无需对现有的代码进行修改就可以对程序进行扩展

在覆盖一个方法时，子类方法不能低于超类方法的可见性
