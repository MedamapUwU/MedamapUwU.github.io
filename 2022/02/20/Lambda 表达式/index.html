Lambda 表达式

何为 lambda 表达式

lambda 表达式是一个可传递的代码块，可以在以后执行一次或多次

为何要引入 lambda 表达式

考虑以下情景：

我们需要对一个字符串数组定制一个比较器 (Comparator) 来完成特定情况的排序，假如我们想按照长度来完成排序而不是默认的方式来完成排序。

上面的方法无法满足我们的要求，在这种情况下，Arrays.sort 方法还有另外一个重载，参数为一个数组和一个比较器 (Comparator)

我们为了实现排序，而将排序方法封装在 compare 方法内，通过 LengthSort 对象来调用这个方法，来达到效果。

实质上，我们需要传的只是 compare 这个代码块，更进一步说是它包含着的 需要传入的参数、进行的具体操作、返回类型

但是 Java 是一种面向对象语言，所以只能创建一个对象，这个对象的类里包含着我们需要的方法

这样是否太过繁琐呢？所以 Java 处理代码段的方法就是引入 lambda 表达式。

Lambda 表达式的形式

参数 （即需要传入的参数）

箭头 (->)

表达式（如果操作无法放在一个表达式中，可以将代码放在 {} 中，并包含显式的 return 8语句），即操作的具体细节

根据参数数量的不同可以有多种变式

  • 没有参数时，仍要提供空括号（如同无参方法一样）
  • 当参数的类型可以被推导出时，可以忽略其类型
  • 如果只有一个参数，且参数类型可被推导出，还可以省略小括号 '()'
  • 方法体只有一句，可以省略大括号
  • 无须指定 lambda 表达式的返回类型
    • lambda 表达式的返回类型总是会由上下文推导得出

如果一个 lambda 表达式只会在某些分支返回一个值，而另外一些分支不返回值，这是不合法的。例如：

(int x) -> {if(x >= 0) return 1;} 没有写 else 的部分的返回值，不合法

前面已经了解了 lambda 表达式的基本用法，那什么情况下使用才合适呢?

函数式接口

定义：对于只有一个抽象方法的接口，需要这种接口的对象时，就可以提供一个 lambda 表达式，这种接口称为 函数式接口

Comparator 接口是一个函数式接口，但是它有两个抽象方法，怎么会是函数式接口呢？

因为尽管 Comparator 有两个抽象方法，但是 equals 方法是 Object 的公共方法(所有的接口都会声明 Object 类的 public 方法) ，因此依然可以被看做 函数式接口

所以看是否为函数式接口时，需要将 Object 中的 public 方法排除

可以通过 @FunctionInterface 注解来确定接口是否为一个函数式接口

换句话说，我们使用 lambda 表达式，需要借助一个载体实现，这个载体就是函数式接口

常用的函数式接口

函数式接口 参数类型 返回类型 抽象方法名 描述 其他方法
Runnable 无 void run 作为无参数或返回值的动作运行
Supplier<T> T get 提供一个 T 类型的值
Consumer<T> T void accept 处理一个 T 类型的值 andThen
BigConsumer<T,U> T,U void accept 处理 T 和 U 类型的值 andThen
Function<T,R> T R apply 有一个 T 类型参数的函数 compose,andThen,identity
BiFunction<T,U,R> T,U R apply 有 T 和 U 类型的参数 andThen
UnaryOperator<T> T T apply 类型 T 上的一元操作符 compose,andThen,identity
BinaryOperator<T> T,T T apply 类型 T 上的二元操作符 andThen,maxBy,minBy
Predicate<T> T boolean test 布尔值函数 and,or,negate,isEqual
BiPredicate<T,U> T,U boolean test 有两个参数的布尔值函数 and,or,negate
