String概况介绍

  • 从概念上讲，Java 字符串就是 Unicode 字符序列

  • 每个用双引号括起来的字符串都是 String 类的实例

  • String 类对象是不可变的 (immutable)，即常量池中一定不存在两个相同的字符串

+ 连接符的浅析

Java 语言允许使用 + 号连接（拼接）两个字符串，其字符串的连接是通过 StringBuilder 的 append 方法实现的。本质是创建了 StringBuilder 对象进行 append 操作，然后将拼接后的 StringBuilder 对象用 toString 方法处理成 String 对象

字符串常量池

  • 介绍
    • 为了防止每次创建相同的字符串浪费空间，有一块区域专门用来存放共享的字符串。
  • 位置
    • 通过查看其他大佬的文章，得出的结论为：
      • 在 JDK6 版本之前，字符串常量池位于 Perm Gen区（也就是方法区）中
      • 在 JDK7 版本之后，字符串常量池被移动到了堆中
  • 使用方法：
    • 直接使用双引号声明出来的 String 对象会直接存放进常量池中
    • 如果不是使用双引号声明的 String 对象，可以使用 String 的 intern 方法。 intern 方法会从字符串常量池中查询当前字符串是否存在，若不存在就会将当前字符串放入常量池。

执行顺序为：

  1. 创建 str1 这个字符串时， JVM 会先去常量池查找是否存在 abc 这个对象。不存在，所以创建一个 abc对象，然后返回 abc这个对象的引用地址给 str1 ，str1 就指向了常量池中的 abc 对象
  2. 创建 str2 这个字符串时， JVM 检查是否存在 abc 对象，存在，所以直接返回 abc 对象的引用地址给 str2
  3. 创建 str3 这个字符串时， 使用的是 new 的方法，所以除了在栈中的 str3 对象，还会在堆中创建一个 obj 对象，str3 中存放的是 obj 的引用地址，JVM 检查存在 abc 对象，将 abc 对象的引用地址返回给 obj

此时：str1 和 str2 中存放的是 abc 的引用地址 ， str3 中存放的是 obj 的引用地址

  1. str1 和 str2 中存放的地址都是 abc 的地址，所以为 true
  2. str1 中存放的是 abc 的引用地址 ， str3 中存放的是 obj 的引用地址，所以为 false

执行顺序为：

  1. 创建 s1 这个字符串时 ，JVM 判断不存在 a 对象，创建，返回 a 的引用地址给 s1
  2. 创建 s2 这个字符串时 ，JVM 判断不存在 b 对象，创建，返回 b 的引用地址给 s2
  3. 创建 s3 这个字符串时，首先创建 StringBuilder 对象进行 append 操作进行拼接，然后通过 toString 方法处理成一个 obj 对象，创建出来，将 obj 的引用地址返回给 s3 ，然后 JVM 判断不存在 ab 对象，创建，返回 ab 的引用地址给 obj

此时： s1 存放的是 a 的引用地址 s2 存放的是 b 的引用地址 s3 存放的是 obj 的引用地址

String.intern() 方法

  • 当不是使用双引号声明 String 对象时（即 使用 new 或者 +），可以使用 String 的 intern 方法。 intern 方法会从字符串常量池中查询当前字符串是否存在，若不存在就会将当前字符串放入常量池。
  • JDK1.7的改动 将 String 常量池 从 Perm 区（方法区）移动到了 Java Heap 区（堆区） String.intern() 方法时，如果存在堆中的对象，会直接保存对象的引用，而不会重新创建对象

区别：

  • 使用 intern() 方法：先去常量池中查询是否存在 abc 对象，找到了就不会创建新的对象；未找到，则创建新对象
  • 不使用 intern() 方法：不会查询是否存在，而是直接创建 abc 对象

综上： 使用 intern() 方法，用时间换取空间。 不使用则用空间换取时间
