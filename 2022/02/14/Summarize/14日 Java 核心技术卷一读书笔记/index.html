14日学习打卡总结

今日范围：

第五章：Object ：所有类的超类 ————》 对象包装器与自动装箱

收获：

Object 类型的变量

可以使用 Object 类型的变量引用任何的对象

在 Java 中，只有基本类型不是对象，例如：数值、字符、布尔类型的值都不是对象

equals 方法

Java 语言规范要求 equals 方法具有下面的特性：

  1. 自反性：对于任何非空引用 x，x.equals(x) 应该返回 true；
  2. 对称性：对于任何引用 x 和 y ，当且仅当 y.equals(x) 返回 true 时，x.equals(y) 返回 true
  3. 传递性：对于任何引用 x 、y、z ，如果 x.equals(x) 返回 true ，y.equals(z) 返回 true 时，x.equals(z) 也返回 true
  4. 一致性：如果 x 和 y 引用的对象没有发生变化，反复调用 x.equals(y) 应该返回同样的结果
  5. 对于任何非空引用 x ，x.equals(null) 应该返回 false

hashCode 方法

散列码(hash code)是由对象导出的一个整型值。散列码是没有规律的。如果 x 和 y 是两个不同的对象， x.hashCode() 与 y.hashCode() 基本不会相同

由于 hashCode 方法定义在 Object 类中，因此每个对象都有一个默认的散列码，其值有对象的存储地址得出

如果重新定义了 equals 方法，就必须为用户可能插入散列表的对象重新定义 hashCode 方法

equals 与 hashCode 方法必须兼容：如果 x.equals(y) 返回值为 true ，那么 x.hashCode() 就必须与 y.hashCode() 返回相同的值。例如：如果定义 Employee.equals()比较员工的 ID ，那么 hashCode 方法就要散列 ID而不是员工的姓名或者存储地址

如果存在数组类型的字段，那么可以使用 静态的 Arrays.hashCode() 方法计算一个散列码，这个散列码有数组元素的散列码组成

泛型数组列表

ArrayList 类类似于数组，但在添加和删除元素时，自动调整数组容量，而不需要自己编写

ArrayList 是一个有 类型参数 (type parameter) 的 泛型类 (generic class),

为了指定数组列表保存的元素对象的类型，需要用一对尖括号将类名括起来追加到 ArrayList 后面 ArrayList<Employee>

使用 add 方法可以将元素添加到数组列表中

如果调用 add 方法，但是内部空间已经满了，数组列表就会自动创建一个更大的数组，将所有的对象 copy 到较大的数组中

如果已经知道或者能够估计出可能存储的元素数量，就可以在 填充数组之前调用 ensureCapacity 方法

数组列表的容量与数组的大小有一个非常重要的区别

size 方法将返回数组列表中包含的实际元素个数

一旦能够确认数组列表的大小将保持恒定，可以使用 trimToSize 方法将存储块的大小调整为当前所需要的存储空间

访问数组列表元素

访问或者改变 ArrayList 对象时，需要使用 get 和 set 方法

只有当数组列表大小 (size) 大于 i 时，才能够调用 list.size(i,x)

对象包装器与自动装箱

有时候需要将 int 这样的基本类型转换为对象。所有的基本类型都有一个与之对应的类,通常这些类称为包装类 wrapper。

包装类是不可变的，一旦构造了包装器，就不允许更改包装在其中的值。同时，包装器还是 final 的，因此不能派生子类

数组列表的 <> 的类型参数不能是基本数据类型，但可以是包装类

由于每个值分别包装对象，所以 ArrayList<Integer> 的效率远远低于 Int[] 数组，只有当方便性 > 执行效率 时，才会考虑这么用

自动装箱这个特性可以很容易的向 ArrayList<Integer> 添加 int 类型的元素。

相反的，当一个 Integer 对象赋给一个 int 值，将会自动拆箱。

基本类型与它们的对象包装器是不一样的，它们有一点很大的不同：同一性。 == 运算符可以用于包装器对象，不过检测的是对象是否有相同的内存位置。

由于包装器引用可以为 null ，所以自动装箱有可能会抛出一个 NullPointerException 的异常

在一个条件表达式中同时使用 Integer 和 Double 的话，Integer 的值会拆箱，提升为 double ，然后装箱为 Double

装箱和拆箱是编译器要做的事，而不是虚拟机

不足：

对于 hashCode 的理解不是很深入，只是了解到了皮毛
