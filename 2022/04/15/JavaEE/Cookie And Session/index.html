会话技术

实现方式：

  • Cookie ：客户端会话跟踪技术，存放在浏览器
  • Session：服务端会话跟踪技术，存放在服务器

Cookie

工作流程

  1. 浏览器发送请求 1 给服务端，服务端接收请求并进行业务处理

  1. 服务端会在处理的过程中创建一个 Cookie 对象，并将浏览器请求的信息放入 Cookie 中

  1. 服务端在响应数据时，会将 Cookie 对象一起相应给浏览器

  1. 浏览器接收到响应数据后，会将 Cookie 对象中的数据存储在浏览器内存中，这时，==浏览器和服务端就建立了一次会话==
  2. 在==同一次会话==中，浏览器再次发送请求 2 给服务端，浏览器会携带 Cookie 对象中的所有数据，服务端在接收到请求和数据后，就可以获取到存储在 Cookie 对象中的所有数据，这样在==同一个会话中==的==多次请求==之间就实现了==数据共享==

基本使用

  1. 创建 Cookie 对象
  1. 通过 response 发送 Cookie
  1. 通过 request 获取 Cookie 数组
  1. 遍历数组

原理

  • 基于 Http 协议的两个请求头信息
    • 响应头：set-cookie
    • 请求头：cookie
  1. 对于服务端相应请求数据时，服务端发现要返回的是一个 Cookie 对象，就会在响应头中添加一行数据

  1. 浏览器获得响应体后，从响应头中获取 Set-Cookie 中的值，储存在浏览器的内存中
  2. 当浏览器再次发送请求给服务端时，浏览器会自动将 Cookie 信息添加到请求头中

  1. Request 对象会把请求头中的 cookie 对象中的值封装成一个个 Cookie 对象，最终形成一个数组

Cookie 存活时间

  • 默认情况下，Cookie 存储在浏览器内存中，当浏览器关闭，内存释放，Cookie 被销毁
  • 要实现 Cookie 持久化存储，要使用 Cookie 提供的 API

Cookie 编码问题

  • 使用 URLEncoder.encode() 方法编码后存入 Cookie 即可
  • 接收端使用 URLEncoder.decode() 进行解码

Session

工作流程

  • 在服务端的 ServletA 获取到一个 Session 对象，将数据放入其中
  • 在服务端的 ServletB 获取到相同的 Session 对象，从中取出数据
  • 这样就可以实现一次会话中多次请求之间的数据共享

基本使用

  1. 获取 Session 对象，使用的是 request 对象

  1. Session 对象提供了许多 API

    • 存储数据到 session 域中
    • 根据 key 获取 value
    • 根据 key，删除该键值对

原理

  • Session 是基于 Cookie 实现的
    • servletA 在第一次获取 Session 对象时，Session 对象会有一个唯一的标识，假设 id = 111
    • 在 Session 对象中存入其他数据并完成业务后，需要通过服务器响应结果给浏览器
    • 服务器发现业务处理中使用了 Session 对象，就会把 Session 的唯一标识 id = 111 当做一个 Cookie ，在响应头中添加 Set-Cookie:JSESSIONID=111 ，并响应给浏览器
    • 在浏览器接收到响应结果后，会把响应头中的 Cookie 数据存储到浏览器的内存中
    • 浏览器在==同一会话==中访问 servletB 时，会把 Cookie 中的数据按照 cookie: JSESSION=111 的格式添加到请求头中并发送给服务器
    • servletB 在接收到请求后，从请求头中获取 cookie 中的 JSESSION 值为 111，然后到服务器中寻找 id = 111 的 Session 对象，如果找到了，直接返回该对象，否则创建一个新的 Session 对象
    • 关闭浏览器后，因为浏览器的默认 cookie 被销毁了，所以就没有 JSESSION 的数据，服务端获取到的 Session 就是一个全新的 Session 对象

Session 钝化与活化

  • 钝化：在服务器正常关闭后，服务器会自动将 Session 数据写入硬盘的文件中
    • 钝化的数据路径为：项目目录\target\tomcat\work\Tomcat\localhost\项目名称\SESSIONS.ser
  • 活化：再次启动服务器后，从文家中加载数据到 Session 中
    • 加载到 Session 后，路径中的 SESSIONS.ser 文件会自动删除

Cookie 与 Session 的区别

  • 存储位置：Cookie 是将数据存储在客户端，Session 将数据存储在服务端
  • 安全性：Cookie 不安全，Session 安全
  • 数据大小：Cookie 最大3KB，Session 无大小限制
  • 存储时间：Cookie 可以通过 setMaxAge() 长期存储，Session默认30分钟
  • 服务器性能：Cookie 不占服务器资源，Session 占用服务器资源

应用场景

Cookie：可以用于登录时”记住我“，第二次登录时不需要输入账号密码
